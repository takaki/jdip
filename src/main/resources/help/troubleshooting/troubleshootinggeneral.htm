General Troubleshooting

I get a StackOverflowError when saving

This is a known problem when saving large maps, such as Youngstown. To fix the problem, increase the heap and (more importantly) stack size that Java needs to run jDip. This may be done using the -Xss512k command-line option. For example: java -Xss512k jdip.jar. If the StackOverFlowError occurs with this setting, try increasing the stack size to 1MB via -Xss1M or -Xss1024k.

Please submit a bug report or email if this does occur with your system configuration.
