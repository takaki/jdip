Order Checking

jDip has two levels of interpreting orders: strict and lenient. This can be changed from the Orders | Order Checking Options... menu item, selecting Order Checking from list that then appears, and changing the option as desired.

Strict order checking, the default, ensures that orders are syntactically valid and legal. For example, a Fleet moving to an adjacent sea or coastal province would be both syntatically valid and legal.

Lenient order checking ensures that orders are syntactically valid, but not nescessarily legal. For example, a Fleet moving to an inland province or non-adjacent space would be a syntactically valid order, however, it would not be a legal order. This may be useful in no-press games, for example.

Note that prior to adjudication, all orders are checked for legality (they always are checked using strict checking). Illegal or invalid orders will fail at that time.

Also note that while orders always must by syntatically valid, locations (Provinces) specified in an order must be found on the map, or jDip will complain.
