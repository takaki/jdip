Entering Orders

Order Entry Methods

Using the Mouse

Orders may be entered using the mouse, by clicking on units or provinces on the map. The currently selected order (e.g., "Move") is shown on the button bar below the map. A different order may be selected by clicking on it.

Orders may be entered by dragging the unit to the desired location, or by clicking on the desired location. The mouse pointer will give visual feedback to indicate if the action is acceptable for the given order.

The statusbar will display both information about the space the mouse pointer is over, and display useful messages about the order entry process.

When the mouse pointer is over the map, the first letter of the order name can be used as a shortcut (e.g., "h" for Hold, "m" for Move, etc.).

Using the Order Box

Orders may be entered by typing or pasting them into the text box, which is located just above the Order List and under the current game phase display. The Enter key must be pressed for the order to be accepted. Orders that are accepted will clear the text box; orders that fail will remain for editing.

If multiple orders need to be entered, the Enter Multiple Orders window (described below) may be used for easier order entry.

Entering Multiple Orders

If multiple orders need to be entered at once, either manually or by pasting them in from another program, the Enter Multiple Orders can be used. This is available from the menu Orders | Enter Multiple Orders... selection.

When entering multiple orders, each order must be on a single line. Orders that span multiple lines, or multiple orders on a single line will cause orders to be misinterpreted.

Order Checking

jDip has two levels of interpreting orders: strict and lenient. This can be changed from the Orders | Order Checking Options... menu item, selecting Order Checking from list that then appears, and changing the option as desired.

Strict order checking, the default, ensures that orders are syntactically valid and legal. For example, a Fleet moving to an adjacent sea or coastal province would be both syntatically valid and legal.

Lenient order checking ensures that orders are syntactically valid, but not nescessarily legal. For example, a Fleet moving to an inland province or non-adjacent space would be a syntactically valid order, however, it would not be a legal order. This may be useful in no-press games, for example.

Note that prior to adjudication, all orders are checked for legality (they always are checked using strict checking). Illegal or invalid orders will fail at that time.

Also note that while orders always must by syntatically valid, locations (Provinces) specified in an order must be found on the map, or jDip will complain.

Changing Orders

Orders may be changed or deleted after they have been entered. To delete an order, or multiple orders, select the order or orders from the Order List and press the "delete" key or select Edit | Delete Selected Order(s).

If a unit already has an order, and a new order is entered for that unit, the new order will automatically replace the old order.

If a mistake is made while entering orders, the Undo and Redo menu options, under the Edit menu, can be used. Remember that Undo and Redo are saved across games, so you can Undo and Redo actions all the way back to the beginning of the game.

Sorting Orders

Orders may be sorted using the buttons at the bottom of the Order List. By clicking a button, orders will be sorted (and stay sorted) by that button. Clicking the same button again will sort the orders in the opposite direction (for example, instead of alphabetical order, reverse-alphabetical order).

A subtle background color will be applied to alternating groups, or alternating rows, as appropriate.

Province sorts the orders alphabetically by Province name
Power sorts the orders alphabetically by the Power name
Unit sorts the orders alphabetically by the Unit name
Order sorts the orders alphabetically by the Order name

Explicit Adjacent Convoys

For armies with a land route and a convoy route, the army is always assumed to move over land. No attempt is made to determine "intent". This is consistent with the DPTG (Diplomacy Players' Technical Guide).

To force an army to travel by convoy, append to a move order the phrase "by convoy" or "via convoy". For example: "A lon-wal by convoy"

The Waive Order

Unused builds do not need to be explicitly waived in jDip. However, some house rules or other adjudicators require builds to be explicitly waived.

jDip fully supports the Waive order. It may be entered via the graphical interface, or by typing in an order (such as "waive build london"). Note that a province is required for a waive order to be accepted. Waive orders are shown as checkmarks over the supply center.
